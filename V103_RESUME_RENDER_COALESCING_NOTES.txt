V103 Resume Render Coalescing

Objective:
Fix the duplicate render burst identified by the V102.1 Diagnostic Log during app resume.

Changes:
- Added a short resume render coalescing window after accepted resume recovery.
- Clears pending layout refresh timers when resume recovery starts.
- Allows one resume render path to run during the resume recovery window.
- Defers canvas-integrity rendering during resume coalescing so it does not race the forced resume render.
- Logs skipped duplicate renders as render-skipped for continued diagnostics.
- Kept Diagnostic Log in place.

Expected result:
Returning from Gallery/Camera, sleep, or another app should no longer trigger canvas-integrity, layout-refresh, and direct renders in the same resume burst.

Preserved:
- Existing UI and kennel layout.
- localStorage compatibility.
- PWA/service worker behavior, with cache version updated to V103.
- Existing diagnostic log access: three-dot menu -> Diagnostic Log -> Copy Log.
