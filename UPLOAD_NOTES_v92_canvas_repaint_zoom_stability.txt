Kennel Manager – v92 canvas repaint + zoom stability

Changes in this version
- Added a canvas compositor reset after closing kennel/incoming modals to address Android/Chrome partial-canvas paint loss where large sections of the kennel board disappear until app restart.
- Changed post-modal Save/Clear rendering to perform a guarded full board render after the modal is gone, instead of only refreshing the edited kennel/tile.
- Removed the inline display:none/display restore modal-close hack from v91, because that can itself contribute to stale compositor layers on Android.
- Added board paint containment/isolation/backface safeguards and a tiny repaint toggle for the kennel canvas.
- Added canvas integrity recovery on app focus/visibility return and orientation changes.
- Tightened pinch/zoom reset behavior so near-1x zoom snaps cleanly back to true home scale.
- Preserved existing kennel data rules, incoming/moved behavior, copy-to-tomorrow behavior, and visual layout choices.

Audit notes
- Updated manifest start_url, service worker registration, service worker cache name, and backup export version to v92.
- JavaScript syntax check passed with Node.
- No intentional changes to kennel count math, PU/DO logic, incoming promotion, or layout defaults.
